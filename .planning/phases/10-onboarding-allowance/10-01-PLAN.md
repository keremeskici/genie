Wire USDC.approve(GenieRouter, budgetAmount) into the onboarding StepBudget flow so that when the user taps "Let's Go", the app sends an on-chain ERC20 approve transaction via MiniKit wallet signing. Without this approval, GenieRouter cannot pull USDC from the user's wallet and no transfers can execute.

Purpose: FOPS-04 (auto-approve threshold executes immediately) and FOPS-06 (GenieRouter handles transfers) both require on-chain allowance to function.
Output: ApprovalOverlay component, contracts constants file, updated onboarding page.

**2. Create `apps/web/src/components/ApprovalOverlay/index.tsx`:**

A full-screen fixed overlay component. Props:
```typescript
interface ApprovalOverlayProps {
  budgetUsd: number;         // e.g. 100
  onSuccess: () => void;     // called after confirmed tx
  onClose: () => void;       // called to dismiss (only in error state for retry reset)
}
```

State machine with 3 states: `'pending' | 'success' | 'error'`.

Implementation (per D-05, D-06, D-07):

- **Pending state:** Full-screen dark overlay (`fixed inset-0 z-50 bg-black/90 flex flex-col items-center justify-center`). Shows an animated spinner (CSS `animate-spin` on a ring div), and text: `"Authorizing Genie to spend up to $${budgetUsd} USDC on your behalf"`. Text uses `text-white/80 text-center text-sm` with `font-headline`.

- **On mount (useEffect):** Immediately call the approval flow:
  1. `const client = createPublicClient({ chain: worldchain, transport: http(process.env.NEXT_PUBLIC_WORLD_CHAIN_RPC_URL!) })`
  2. `const { poll } = useUserOperationReceipt({ client })` — NOTE: since this is a hook, initialize it at component top level, not inside useEffect.
  3. Calculate amount: `BigInt(budgetUsd) * BigInt(1_000_000)` (USDC has 6 decimals, per D-03).
  4. Call `MiniKit.sendTransaction({ chainId: 480, transactions: [{ to: USDC_ADDRESS, data: encodeFunctionData({ abi: ERC20_APPROVE_ABI, functionName: 'approve', args: [GENIE_ROUTER_ADDRESS, amount] }) }] })`.
  5. `await poll(result.data.userOpHash)`.
  6. On success: set state to `'success'`, wait 1.5 seconds (`setTimeout`), then call `onSuccess()`.

- **Success state:** Same overlay, spinner replaced with a green checkmark (use `text-accent` color `#ccff00`), text changes to `"Approved! Redirecting..."`.

- **Error state:** Same overlay, show error icon, text `"Transaction failed or was rejected"`, and a neon-accent "Try Again" button (`bg-[#ccff00] text-black font-headline font-bold rounded-2xl px-8 py-4`). Try Again resets state to `'pending'` and re-runs the approval flow.

- Wrap the sendTransaction + poll sequence in try/catch. On catch, set state to `'error'`.

- Use `'use client'` directive at top.

- Imports: `MiniKit` from `@worldcoin/minikit-js`, `useUserOperationReceipt` from `@worldcoin/minikit-react`, `createPublicClient`, `encodeFunctionData`, `http` from `viem`, `worldchain` from `viem/chains`, `useState`, `useEffect`, `useCallback` from `react`, and `ERC20_APPROVE_ABI`, `USDC_ADDRESS`, `GENIE_ROUTER_ADDRESS` from `@/lib/contracts`.
