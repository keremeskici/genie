## Cross-Phase Integration Report

### Critical Issues

**1. ConfirmCard URL Missing `/api` Prefix (CRITICAL)**
- **File:** `apps/web/src/components/ConfirmCard/index.tsx:51`
- **Bug:** Fetches `${API_URL}/confirm` instead of `${API_URL}/api/confirm`
- **Impact:** Every over-threshold transfer confirmation returns 404. FOPS-05 broken end-to-end.
- **Fix:** Change `/confirm` to `/api/confirm` in the fetch URL

**2. ProfileInterface Spending Limit Not Persisted (HIGH)**
- **File:** `apps/web/src/components/ProfileInterface/index.tsx:21-27`
- **Bug:** `handleSaveLimit` sets local state only with a TODO comment. Never calls `PATCH /api/users/profile`.
- **Impact:** Users cannot update their auto-approve threshold from the Profile page. Only the onboarding value persists.
- **Affects:** FOPS-04, FOPS-05

**3. Backend Routes Missing Auth Guards (HIGH)**
- **Files:** `apps/api/src/routes/confirm.ts`, `apps/api/src/routes/users.ts`
- **Bug:** `/api/confirm` and `/api/users/profile` accept `userId` from request body without session validation.
- **Impact:** Any caller who guesses a valid txId+userId can execute transfers or overwrite thresholds.
- **Affects:** WRID-03, WRID-04

### Moderate Issues

**4. System Prompt Missing `contact_list` JSON Format (MODERATE)**
- **File:** `apps/api/src/prompts/system.md`
- **Bug:** System prompt instructs `confirmation_required` JSON format but not `contact_list` format.
- **Impact:** Contact disambiguation UI in ChatInterface may not render consistently.
- **Affects:** FOPS-03, MAPP-04

**5. Verify Component Missing `onVerified` Callback (MODERATE)**
- **File:** `apps/web/src/components/Verify/index.tsx`
- **Bug:** `export const Verify = ()` — zero props, no callback interface.
- **Impact:** Parent components cannot react to verification success. WRID-01 incomplete.
